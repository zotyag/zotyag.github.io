1. Java kód fordításának lépései

  1. Írás: A Java kódot .java kiterjesztésű fájlban írjuk meg.
  2. Fordítás: A javac parancs segítségével lefordítjuk a kódot bájtkódra (.class fájl).
  3. Futtatás: A java parancs segítségével a JVM (Java Virtual Machine) végrehajtja a .class fájlt.

2. JVM: mi az, miért fontos?

  • Mi az?
    A Java Virtual Machine egy futtatókörnyezet, amely a Java programokat bájtkódból futtatható kóddá alakítja a gépen.
  • Miért fontos?
    • Platformfüggetlenséget biztosít, mivel a bájtkód minden JVM-en ugyanúgy fut.
    • Automatizált memóriakezelést (Garbage Collection) végez.

3. JRE, JDK, IDE: mire lehet használni?

  • JRE (Java Runtime Environment):
    A Java programok futtatásához szükséges környezet. Tartalmazza a JVM-et és a Java könyvtárakat.
  • JDK (Java Development Kit):
    A Java programok fejlesztéséhez és futtatásához szükséges eszközkészlet. Tartalmazza a JRE-t, fordítót, és más fejlesztési eszközöket.
  • IDE (Integrated Development Environment):
    Olyan eszköz, mint az IntelliJ IDEA vagy Eclipse, amely segíti a Java programok fejlesztését kódszerkesztő, hibakereső, és más funkciók biztosításával.

4. main metódus: hogy néz ki, miért fontos, mi a szerepe?

  • Hogy néz ki?
    public static void main(String[] args) {
    // A program belépési pontja
}
  • Miért fontos?
    Ez a program belépési pontja. A JVM innen kezdi a kód végrehajtását.
  • Szerepe:
    A program futásának elindítása, a kezdő logika meghatározása.

6. Az objektum és osztály definíciója

  • Objektum:
    Egy osztály példánya, amely tartalmazza az adott osztály attribútumainak (állapot) és metódusainak (viselkedés) megvalósítását. Például: egy „Autó” osztályból létrehozott konkrét „piros autó”.
  • Osztály:
    Egy sablon vagy tervrajz, amely meghatározza az objektumok attribútumait (adattagok) és viselkedését (metódusok).

7. Absztrakció fogalma

  • Az absztrakció az a folyamat, amikor egy rendszer bonyolultságát csökkentjük azáltal, hogy csak a lényeges tulajdonságokat emeljük ki, és elrejtjük a részleteket.
  • Példa: Egy autó vezetéséhez nem kell tudni, hogyan működik a motor, csak azt, hogyan kell használni a kormányt és a pedálokat.

8. Mit jelent az egységbezárás (encapsulation)?

  • Az egységbezárás az objektumok adatainak és metódusainak védelme oly módon, hogy azok csak az osztály által meghatározott módokon legyenek elérhetők.
  • Java-ban megvalósítás:
    • Adattagokat private kulcsszóval zárunk el.
    • Getterekkel és setterekkel biztosítjuk a hozzáférést.

9. Mi a különbség az objektum állapota és viselkedése között?

  • Állapot:
    Az objektum adattagjai által tárolt értékek. Példa: egy „Autó” objektum színe, sebessége, gyártási éve.
  • Viselkedés:
    Az objektum által végrehajtott műveletek (metódusok). Példa: egy „Autó” gyorsítása vagy fékezése.

10. Mi az osztályattribútum és osztálymetódus?

  • Osztályattribútum:
    Olyan adattag, amely az osztályhoz kötődik, és minden példány számára közös. Statikus kulcsszóval (static) jelöljük.
    • Példa: public static int counter;
  • Osztálymetódus:
    Olyan metódus, amely az osztályhoz tartozik, nem egy adott példányhoz. Statikus kulcsszóval (static) jelöljük.
    • Példa: public static void printCounter();

11. Mi az a getter és setter?

  • Getter:
    Egy metódus, amely visszaadja egy privát adattag értékét.
    public int getAge() {
    return age;
}
  • Setter:
    Egy metódus, amely beállítja egy privát adattag értékét.
    public void setAge(int age) {
    this.age = age;
}
  • Szerepük: Az egységbezárás megvalósítása és az adattagok biztonságos elérése.

12. Mi az öröklés (általánosan)?

  • Az öröklés az objektumorientált programozás egyik alapelve, amely lehetővé teszi, hogy egy osztály (gyerekosztály) örökölje egy másik osztály (szülőosztály) attribútumait és metódusait.
  • Célja: Kód újrafelhasználás, strukturált programozás.

13. Mi az aggregáció? Mi a kompozíció? Mi az asszociáció?

  • Asszociáció: Két osztály közötti általános kapcsolat.
    • Példa: Egy „Tanár” tanít „Diákokat”.
  • Aggregáció: Gyenge kapcsolat, ahol az egyik objektum egy másik része, de függetlenül létezhet.
    • Példa: Egy „Osztály” tartalmaz „Diákokat”, de a diákok önállóan is létezhetnek.
  • Kompozíció: Erős kapcsolat, ahol az egyik objektum a másik része, és nem létezhet nélküle.
    • Példa: Egy „Autó” és annak „Motorja”.

14. Mi az absztrakt osztály?

  • Egy olyan osztály, amelyet nem lehet példányosítani, csak leszármaztatni.
  • Tartalmazhat absztrakt (implementáció nélküli) és konkrét metódusokat is.
    abstract class Shape {
    abstract void draw(); // absztrakt metódus
}

16. Névkonvenciók Java-ban

  • Osztályok: PascalCase formátum, pl. MyClass.
  • Adattagok és metódusok: camelCase formátum, pl. myVariable, myMethod().
  • Konstansok: Nagybetűk és aláhúzásjelek, pl. MAX_VALUE.
  • Paraméterek: camelCase formátum, pl. parameterName.

17. Mi az a konstruktor? Mi történik, ha nem adunk meg konstruktort?

  • Konstruktor: Olyan speciális metódus, amelyet egy osztály példányosításakor hív meg a JVM.
    • Szintaxis: Az osztály nevét viseli, nincs visszatérési típusa.
    public MyClass() {
    // Inicializálás
}
  • Ha nincs megadva: A Java fordító egy alapértelmezett üres konstruktort generál.

18. Hogyan példányosítunk Java-ban egy osztályt?

MyClass obj = new MyClass();

19. Java Garbage Collector mit csinál?

  • A Garbage Collector (GC) automatikusan felszabadítja azokat az objektumokat a memóriából, amelyekre már nincs hivatkozás.
  • Fontossága: Megakadályozza a memória túlcsordulását és optimalizálja az erőforrás-kezelést.

20. Osztály tagjainak és metódusainak láthatósági módosítói

  1. public: Bárhonnan elérhető.
  2. private: Csak az adott osztályból elérhető.
  3. protected: Csak ugyanabban a csomagban vagy leszármazott osztályokban elérhető.
  4. Alapértelmezett (package-private): Csak ugyanabban a csomagban elérhető.

21. A static kulcsszó használata Java-ban

  • Adattagoknál: Az osztály szintjén osztozik minden példány.
  • Metódusoknál: Az osztály szintjén érhető el példányosítás nélkül.
  • Blokkoknál: Egyszer fut le, amikor az osztály betöltődik.

22. Hogyan deklarálunk konstanst? Névkonvenció?

  • Deklaráció: final kulcsszóval.
    public static final int MAX_VALUE = 100;
  • Névkonvenció: Nagybetűk és aláhúzások.

23. Mit csinál a final kulcsszó?

  • Osztályok: Megakadályozza az osztály öröklését.
  • Metódusok: Nem lehet felülírni.
  • Adattagok: Nem lehet újra inicializálni.

24. Tömb (array) deklarációja és használata

  • Deklaráció:
    int[] numbers = new int[5];
  • Használat:
    numbers[0] = 10; // Értékadás
System.out.println(numbers[0]); // Érték kiíratása

25. Mi az a Java csomag? Hogyan adunk neki nevet?

  • Csomag: Olyan névtér, amely osztályokat és interfészeket csoportosít.
  • Deklarálás:
    package mypackage;

26. Csomag elemeinek láthatósági módosítói

  • public: Bárhonnan elérhető.
  • Alapértelmezett: Csak ugyanabban a csomagban elérhető.

27. Hogyan lehet használni a csomag elemeit?

  • Importálás:
    import mypackage.MyClass;
  • Teljes név:
    mypackage.MyClass obj = new mypackage.MyClass();

29. Mi az az annotáció?

  • Annotáció: Kiegészítő információt biztosító metaadatok a kódban.
  • Formátum:
    @Override
public void myMethod() {}

30. Mit csinál az @Override annotáció?

  • Ellenőrzi, hogy a metódus felülír egy szülőosztályban lévő metódust.

31. Mit csinál a this kulcsszó?

  • Az aktuális objektumra hivatkozik.
  • Használat:
    • Az adattagok elérése.
    • Konstruktorhívás láncolása.

32. Mit csinál a super kulcsszó?

  • A szülőosztályra hivatkozik.
  • Használat:
    • Szülőosztály konstruktorának hívása.
    • Szülőosztály metódusainak elérése.

33. Miért fontos a Java Object osztálya?

  • A Java minden osztályának szülőosztálya.
  • Alapmetódusokat biztosít, pl. toString(), equals(), hashCode().

34. Mi a baj a String osztállyal? Mit és hogyan használunk helyette?

  • String osztály problémája:

    • A String immutábilis, ami azt jelenti, hogy minden módosítás új objektumot hoz létre. Ez memória- és teljesítményproblémákat okozhat.

  • Alternatíva:

    • StringBuilder: Nem szálbiztos, de gyorsabb.
    • StringBuffer: Szálbiztos, de lassabb.

  • Használat StringBuilder-rel:

    StringBuilder sb = new StringBuilder("Hello");
sb.append(" World");
System.out.println(sb.toString());

35. Numerikus típusok közötti konverzió Java-ban

  • Automatikus típuskonverzió (widening):
    Kisebb típus automatikusan konvertálódik nagyobbra.

    int i = 10;
double d = i; // Automatikus konverzió

  • Kényszerített típuskonverzió (casting):
    Nagyobb típusból kisebbbe kényszerítéssel.

    double d = 10.5;
int i = (int) d; // Kényszerített konverzió

36. Hogyan konvertálunk számot Stringgé és vissza?

  • Számból String:

    int num = 123;
String str = String.valueOf(num);

  • Stringből szám:

    String str = "123";
int num = Integer.parseInt(str);

37. Java paraméterátadás módja

  • Érték szerint: A Java mindig érték szerint adja át a paramétereket.
    • Primitív típusok: A paraméter értéke másolódik.
    • Objektumok: Az objektum referencia másolódik, de az objektum állapota módosítható.

38. Hogyan lehet tetszőleges számú paramétert átadni egy metódusnak?

  • Varargs (változó hosszúságú paraméterlista):

    public void printNumbers(int... numbers) {
    for (int num : numbers) {
        System.out.println(num);
    }
}

  • Hívás:

    printNumbers(1, 2, 3, 4, 5);

39. Mit csinál Java-ban a return utasítás?

  • Visszaad egy értéket a hívó metódusnak.
  • Használat:
    public int add(int a, int b) {
    return a + b;
}

40. Inicializáló mező és blokk Java-ban

  • Inicializáló mező: Adattagok deklaráláskor történő inicializálása.

    int x = 10;

  • Inicializáló blokk: Az összes példányosításkor lefutó kód.

    static {
    System.out.println("Inicializáló blokk fut");
}

41. Java Enum típus

    public enum Day {

42. Java osztályok közötti öröklés

    // Szülőosztály tartalma
}

public class Child extends Parent {

  • A gyermekosztály konstruktora explicit vagy implicit hívja a szülőosztály konstruktorát a super kulcsszóval.

44. Metódus felülírás öröklés esetén

  • A gyermekosztály újradefiniálhatja a szülőosztály metódusát.
  • Szabályok:
    • Azonos aláírás.
    • Nem lehet szigorúbb láthatóságú.
    @Override
public void parentMethod() {
    // Új implementáció
}

45. Ugyanolyan nevű statikus metódus a szuperosztályban és alosztályban

  • Statikus metódusok nem öröklődnek, és nem lehet őket felülírni, csak elrejteni.

    • Szuperosztály metódusa hívható az osztály nevével:
      Parent.staticMethod();

  • Ha csak a szuperosztály definiál statikus metódust, az alosztályon keresztül is hívható:

    Child.staticMethod(); // Hívja a szuperosztály metódusát

46. Ugyanolyan nevű adattag a szuper- és alosztályban

  • Az adattagok elérhetősége az objektum referencia típusától függ.
    • Szuperosztály adattagjának elérése:
      super.attributeName;

47. Konstruktorok lánca (Chain of Constructors)

  • Konstruktor egy másik konstruktor hívása ugyanabban az osztályban a this kulcsszóval.
    public MyClass() {

}

public MyClass(int value) {
    // Inicializálás
}

48. Láthatósági módosítók változása metódus felülíráskor

  • Szigorúbb láthatóság nem megengedett.
    • Példa: A szülőosztály protected metódusa csak protected vagy public lehet az alosztályban.

49. Polimorfizmus Java-ban

  • Fogalom: Metódusok felülírása, egy osztályon belül több azonos nevű metódus vagy azonos nevű metódusok szülő és gyerek osztályban.
    • Metódusok felülírása:
      Parent obj = new Child();
obj.method(); // A Child implementációját hívja

50. Absztrakt osztály és absztrakt metódus

    public abstract class Shape {

52. Mi az az UML?

  • Unified Modeling Language (UML): Egy vizuális modellezési nyelv, amelyet szoftverek és rendszerek tervezéséhez használnak.

53. Hogy néz ki az UML osztálydiagram?

  • Alap elemek:
    • Osztályok: Téglalapok, amelyek tartalmazzák az osztály nevét, attribútumait és metódusait.
    • Kapcsolatok: Vonallal ábrázolt kapcsolatok (asszociáció, aggregáció, kompozíció, öröklés).

54. Az UML hogyan jelöli az osztályok és interfészek közötti öröklést?

  • Az osztály és interfész közötti öröklés egy üres háromszög alakú nyíllal jelölt vonal.
  • Egy osztály vagy interfész több interfészt is örökölhet.

55. Az UML hogyan jelöli az absztrakt osztályt és interfészt?

  • Absztrakt osztály: Az osztály neve dőlt betűvel szerepel.
  • Interfész: Téglalapban a <<interface>> kulcsszóval van megjelölve.

56. Mi az a Java interfész? Mit tartalmazhat?

  • Egy absztrakt típus, amely:
    • Absztrakt metódusokat
    • Default metódusokat
    • Statikus metódusokat
    • Konstansokat tartalmazhat.

57. Mire szolgál a Java interfész default metódusa?

  • Funkciója: Lehetővé teszi, hogy az interfész új metódusokat vezessen be anélkül, hogy a meglévő implementációkat módosítani kellene.

58. Java interfészek közötti öröklés, interfész és osztály közötti öröklés

  • Interfészek közötti öröklés: Egy interfész több másik interfészt is örökölhet.

    public interface A { }
public interface B extends A { }

  • Interfész és osztály közötti öröklés: Egy osztály implementálhat egy vagy több interfészt.

    public class MyClass implements A, B { }

59. Hogyan lehet használni egy Java interfészt?

  • Egy osztálynak implementálnia kell az interfész összes absztrakt metódusát.
    public interface A {
    void method();
}

public class MyClass implements A {
    public void method() {
        System.out.println("Implemented method");
    }
}

60. Default metódusok konfliktusának feloldása interfészek öröklése során

  • Ha két interfész azonos nevű default metódust tartalmaz, az implementáló osztálynak felül kell írnia a metódust, és egyértelműsítenie kell, melyik implementációt használja.
    interface A {
    default void method() {
        System.out.println("A");
    }
}

interface B {
    default void method() {
        System.out.println("B");
    }
}

public class MyClass implements A, B {
    public void method() {
        A.super.method();
    }
}

61. Comparable interfész

  • Összehasonlítási logikát definiál az compareTo metódussal.
    public class MyClass implements Comparable<MyClass> {
    public int compareTo(MyClass other) {
        return this.value - other.value;
    }
}

62. Comparator interfész

  • Összehasonlítást biztosít egyedi logikával az compare metódus segítségével.
    import java.util.Comparator;

public class MyComparator implements Comparator<MyClass> {
    public int compare(MyClass o1, MyClass o2) {
        return o1.value - o2.value;
    }
}

63. Mi az a Java generikus? Hogyan definiálhatunk generikusokat?

  • Generikus: Típusbiztos, újrafelhasználható kódot tesz lehetővé.
  • Definíció:
    public class GenericClass<T> {
    private T data;
    public T getData() { return data; }
    public void setData(T data) { this.data = data; }
}

64. Java generikus metódus, statikus metódus, és hívásuk

  • Generikus metódus:

    public static <T> void printArray(T[] array) {
    for (T element : array) {
        System.out.println(element);
    }
}

    Hívás:

    String[] strArray = {"A", "B", "C"};
printArray(strArray);

  • Statikus generikus metódus: Statikus metódusban is használható generikus típus.

    public static <T> T getFirstElement(T[] array) {
    return array[0];
}

65. Java generikus, bounded type

  • Bounded type: Generikus típus megkötése.
    public class BoundedType<T extends Number> {
    private T value;
    public T getValue() { return value; }
    public void setValue(T value) { this.value = value; }
}
    Itt a T csak Number vagy annak leszármazottja lehet.

66. Java generikus, wildcards

  • Wildcards: ? szimbólummal általánosítja a generikus típusokat.

    public void printList(List<?> list) {
    for (Object obj : list) {
        System.out.println(obj);
    }
}

  • Bounded wildcards:

    public void printList(List<? extends Number> list) {
    for (Number num : list) {
        System.out.println(num);
    }
}

67. Java generikus, type erasure

  • Type erasure: A generikus típusok fordítás közben eltávolításra kerülnek, és helyükre az alapértelmezett (általában Object) típus lép. Ez biztosítja a visszafelé kompatibilitást.

68. Mi az a kollekció? Deklaráljon és példányosítson egyet.

  • Kollekció: Adatok gyűjteménye, amely rugalmas méretet és különböző típusú elemek tárolását teszi lehetővé.

    Példa:

    List<String> list = new ArrayList<>();
list.add("Elem1");
list.add("Elem2");

69. A kollekció elemeinek a rendezését hogyan lehet megvalósítani?

  • Comparator vagy Comparable interfész:
    Collections.sort(list, new Comparator<String>() {
    public int compare(String o1, String o2) {
        return o1.compareTo(o2);
    }
});

70. Collections osztály

  • Funkció: Segédmetódusokat biztosít kollekciók kezelésére, például:
    • sort(List list) - rendezés
    • reverse(List list) - lista fordítása
    • shuffle(List list) - elemek keverése
    • min(Collection coll) - legkisebb elem

71. Rajzolja le a Java kollekciók interfészeinek és osztályainak öröklési rendszerét!

Leírás:

  • Interfészek:

    • Collection (szülő interfész) → List, Set, Queue
    • Map nem része a Collection hierarchiának, de szorosan kapcsolódik hozzá.

  • Implementációk:

    • List: ArrayList, LinkedList, Vector
    • Set: HashSet, TreeSet, LinkedHashSet
    • Queue: PriorityQueue, Deque

72. Hogyan különbözteti meg a HashSet az elemeket egymástól?

  • A hashCode() és equals() metódusok segítségével ellenőrzi az elemek egyediségét.

73. Hogyan különbözteti meg a TreeSet az elemeket egymástól?

  • Az elemeket természetes sorrendjük vagy egyedi Comparator alapján rendezi.

74. Java beágyazott osztály

  • Egy osztály egy másik osztályon belül definiált.
    • Non-static beágyazott osztály (inner class): Hozzáfér a külső osztály példányának tagjaihoz.

75. Statikus beágyazott osztály

  • Static nested class: Csak a külső osztály statikus tagjaihoz fér hozzá.
    public class Outer {
    static class Nested {
        public void display() {

        }

}

76. Java local class

  • Egy metóduson vagy blokkon belül definiált osztály, csak lokálisan érhető el.
    public void method() {

    LocalClass lc = new LocalClass();

}

80. Java lambda kifejezés

  • Egy rövidített szintaxis egy funkcionális interfész implementálására.
    List<String> list = Arrays.asList("A", "B", "C");
list.forEach(item -> System.out.println(item));

81. Java metódus referencia

  • Metódus referencia: A lambda kifejezések rövidebb alternatívája, amikor a metódus neve önmagában is elegendő.
    Típusai:

    1. Statikus metódus referencia:

      Consumer<String> printer = System.out::println;
printer.accept("Hello, World!");

    2. Példány metódus referencia:

      String str = "example";
Supplier<Integer> length = str::length;
System.out.println(length.get());

    3. Konstruktor referencia:

      Supplier<List<String>> listSupplier = ArrayList::new;
List<String> list = listSupplier.get();

82. Mi az a kivétel?

  • Kivétel (Exception): Egy futásidejű hiba vagy rendellenesség, amely megszakíthatja a program végrehajtását.

83. Hogyan lehet kezelni a kivételt?

  • A kivételek a try-catch blokkal kezelhetők:
    try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Hiba: " + e.getMessage());
}

84. A kivételeknek mi a 3 alapvető kategóriája?

  1. Checked Exceptions: Kötelezően kezelendők, például: IOException.
  2. Unchecked Exceptions: Nem kötelezően kezelendők, például: NullPointerException.
  3. Error: Súlyos hibák, amelyeket nem lehet kezelni, például: OutOfMemoryError.

85. Hogy néz ki a try-catch-final utasítás, és melyik része mit csinál?

  • try blokk: Kód, amely hibát okozhat.
  • catch blokk: A kivétel kezelése.
  • finally blokk: Mindig lefut, akár történt kivétel, akár nem.
    try {
    int num = Integer.parseInt("abc");
} catch (NumberFormatException e) {
    System.out.println("Hiba történt!");
} finally {
    System.out.println("A program folytatódik.");
}

86. Mit csinál a try-with-resource utasítás?

  • Automatikusan bezárja a megnyitott erőforrásokat (pl. fájlokat, adatbázis-kapcsolatokat).
    try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    System.out.println(br.readLine());
} catch (IOException e) {
    e.printStackTrace();
}

87. Hogyan lehet kivételt dobni?

  • A throw kulcsszóval dobhatunk kivételeket.
    public void validate(int age) {
    if (age < 18) {
        throw new IllegalArgumentException("Nem elég idős.");
    }
}

88. Hogyan hozhatunk létre Java kivétel osztályokat?

  • Egyedi kivételosztály létrehozása:
    public class CustomException extends Exception {
    public CustomException(String message) {
        super(message);
    }
}

89. Mi az I/O stream? Mi lehet a forrása és célhelye az I/O streameknek?

  • I/O stream: Adatok folyamatos átvitelét jelenti egy forrás (pl. fájl) és cél (pl. konzol) között.

90. Mi az a byte stream?

  • Byte stream: Bináris adatokat olvas vagy ír (pl. képek, hangok).
    Példa: FileInputStream, FileOutputStream.

91. Mi az a character stream?

  • Character stream: Szöveges adatokat kezel (karakterek olvasása és írása).
    Példa: FileReader, FileWriter.

92. Mi az a buffered stream?

  • Buffered stream: Gyorsítótárat használ az I/O műveletek hatékonyságának növelésére.
    Példa: BufferedReader, BufferedWriter.

93. Scanning and Formatting

  • Scanning: Adatok beolvasása, például:
    Scanner sc = new Scanner(System.in);
int num = sc.nextInt();
  • Formatting: Formázott kimenet készítése:
    System.out.printf("Szám: %d%n", num);

94. Mi a 3 standard stream a Java-ban? Melyik mire való?

  1. System.in: Standard bemenet (pl. billentyűzet).
  2. System.out: Standard kimenet (pl. konzol).
  3. System.err: Hibakimenet.

95. Mi a data stream?

  • Ez egy funkcionális programozási eszköz, amely lehetővé teszi az adatok feldolgozását egy adatfolyamként (stream). A Java stream nem tárolja az adatokat, hanem egy adatforrás (például gyűjtemények, tömbök vagy I/O csatornák) elemeit dolgozza fel sorban, miközben különböző műveleteket végez rajtuk.

  • Típusok: Objektumok Stream-je, Primitívek Stream-je

  • Műveletek streameken:
    1. Intermediate (közbenső) műveletek pl.: filter(Predicate)
    2. Terminal (végső) műveletek pl.: count()

96. Mi az object stream?

  • Az objektum stream a Java Stream API általános típusa, amely bármilyen objektumot tartalmazhat. Ez a leggyakrabban használt stream típus, mivel lehetővé teszi a Java-ban található összetett objektumok (pl. String, Integer, ...) feldolgozását.
    Példa:
    
    List <String> names = Arrays.asList("Alice", "Bob", "Charlie", "Anna");
    names.stream().filter(name -> name.startsWith("A"))

97. Mi a tesztelés? Miért fontos?

  • A program helyességének ellenőrzése. Fontos a hibák megelőzése és a megbízhatóság növelése érdekében.

98. Milyen teszttípusokról volt szó előadáson? Melyik mit csinál?

  1. Unit teszt: Egyedi metódusokat/osztályokat tesztel elszigetelten.
  2. Integrációs teszt: Az összekapcsolt modulok helyes együttműködését vizsgálja.
  3. Funkcionális teszt: Az alkalmazás funkcióinak megfelelő működését ellenőrzi.
  4. Teljesítményteszt: Az alkalmazás sebességét, terhelhetőségét méri.
  5. Rendszerteszt: Az egész rendszer megfelelő működését vizsgálja.

99. Mire való a JUnit?

  • A JUnit egy Java tesztkeretrendszer, amely unit tesztek automatizált futtatására szolgál.
  • Segítségével gyorsan ellenőrizhető a kód helyessége.

100. Soroljon fel a JUnit eszközben használatos annotációkból ötöt és magyarázza mire valók!

  1. @Test: Egy metódust tesztként jelöl.
  2. @BeforeEach: A tesztek előtt futó inicializáló kód.
  3. @AfterEach: A tesztek után futó tisztítási kód.
  4. @BeforeAll: Az összes teszt előtt egyszer lefutó kód.
  5. @AfterAll: Az összes teszt után egyszer lefutó kód.

101. A JUnit eszköz assertation-jai mire valók? Soroljon fel belőlük 10-et és magyarázza őket!

Az assert-ek a tesztelés során elvárt eredmények ellenőrzésére szolgálnak:

  1. assertEquals(expected, actual): Két érték egyenlőségét ellenőrzi.
  2. assertNotEquals(unexpected, actual): Ellenőrzi, hogy két érték nem egyenlő.
  3. assertTrue(condition): Ellenőrzi, hogy a feltétel igaz.
  4. assertFalse(condition): Ellenőrzi, hogy a feltétel hamis.
  5. assertNull(object): Ellenőrzi, hogy az objektum null.
  6. assertNotNull(object): Ellenőrzi, hogy az objektum nem null.
  7. assertSame(expected, actual): Ellenőrzi, hogy két objektum ugyanaz.
  8. assertNotSame(unexpected, actual): Ellenőrzi, hogy két objektum különböző.
  9. assertThrows(Exception.class, () -> { ... }): Ellenőrzi, hogy egy kódrészlet kivételt dob.
  10. assertTimeout(Duration.ofSeconds(1), () -> { ... }): Ellenőrzi, hogy egy művelet adott idő alatt lefut.

102. Javadoc mit csinál, mire való, miért használjuk?

  • A Javadoc automatikusan dokumentációt generál a kódhoz speciális kommentekből.
  • Fontos a kód érthetőségének és fenntarthatóságának növeléséhez.

103. Javadoc eszköz használata esetén a fő leírást hova írja? Milyen elemekhez adhat meg dokumentációt?

  • A fő leírás az osztály, interfész, metódus vagy változó elé kerül.
  • Dokumentálható elemek: osztályok, interfészek, metódusok, attribútumok.

104. A metódus dokumentációjának (javadoc) mik a kötelező elemei? Magyarázza az egyes elemeket.

  1. @param: A metódus paramétereinek leírása.
  2. @return: A visszatérési érték leírása.
  3. @throws: A metódus által dobott kivételek leírása.

105. A metódus dokumentációjának kötelező elemein kívül soroljon fel és magyarázzon 5 standard tag-et (javadoc)!

  1. @author: A kód írója.
  2. @version: A kód verziója.
  3. @see: Hivatkozás más osztályra vagy metódusra.
  4. @since: A funkció bevezetésének verziója.
  5. @deprecated: Jelzi, hogy az elem elavult.

106. A JAR eszköz mire való, milyen műveleteket lehet vele elvégezni?

  • Java Archívum (JAR) fájlok létrehozására és kezelésére szolgál. Ez egy tömörített archívum, amely Java osztályfájlokat, erőforrásokat (például képeket, hangfájlokat), metaadatokat és opcionálisan egy manifest fájlt tartalmaz. A JAR fájlok célja az alkalmazások és könyvtárak egyszerűbb terjesztése, telepítése és futtatása.
  • Használható csomagolásra, tömörítésre, futtatható fájlok készítésére.

107. A JAR file „manifest”-jét mutassa be!

  • Egy META-INF/MANIFEST.MF fájlban tárolt információkat(metaadatokat) tartalmaz (pl. fő osztály, osztályok listája).
    Példa:
    Manifest-Version: 1.0
Main-Class: com.example.Main

108. Mi az a Java modul?

  • A Java modul egy logikai egység, amely osztályokat és csomagokat csoportosít.
  • Java 9-től érhető el.

109. A Java modul module-info.java állományában mit jelentenek az exports, exports … to, requires, uses direktívák?

  1. exports: Csomagok láthatóvá tétele más modulok számára.
  2. exports ... to: Csomagok exportálása konkrét moduloknak.
  3. requires: Függőség egy másik modulra.
  4. uses: Szolgáltatás használatát jelzi.

110. A kollekciók aggregáló műveletei esetén a pipeline-nak milyen részei vannak? Melyik mire való?

  1. Forrás: Pl. lista, tömb.
  2. Közbenső műveletek: Pl. szűrés, rendezés (nem végrehajtott).
  3. Terminális művelet: Pl. gyűjtés, kiírás (végrehajtott).

111. Hogyan működik a reduce művelet a kollekciók aggregáló műveletei esetén?

  • Egyetlen eredményt hoz létre az elemek kombinálásával.
    int sum = list.stream().reduce(0, Integer::sum);

112. Hogyan működik a collect művelet a kollekciók aggregáló műveletei esetén?

  • Az elemeket egy új kollekcióba gyűjti:
    List<String> collected = stream.collect(Collectors.toList());

113. Mutassa be a kollekciók aggregáló műveleteiből a groupingBy és reducing műveleteket!

  • groupingBy: Csoportosítást végez:

    Map<String, List<Person>> grouped = people.stream()
    .collect(Collectors.groupingBy(Person::getCity));

  • reducing: Egyedi kombinációkat végez:

    Optional<Integer> sum = stream.collect(Collectors.reducing(Integer::sum));

    • 201. Mi az a Project Lombok?

    • A Project Lombok egy Java könyvtár, amely egyszerűsíti a kódot azáltal, hogy automatikusan generál gettereket, settereket, toString-et, equals-t és más metódusokat fordítási időben.
    • Segítségével csökkenthető a boilerplate kód, így tisztább és karbantarthatóbb lesz a program.

    202. Project Lombok-kal hogyan ad meg gettert, settert?

    • A getterekhez és setterekhez az osztály adattagjai fölé helyezett @Getter és @Setter annotációkat használjuk.
    • Példa: 
      @Getter
@Setter
private String name;

    203. Project Lombok-kal hogyan ad meg toString-et?

    • A @ToString annotációval automatikusan generálható egy toString metódus az osztályhoz.
    • Példa: 
      @ToString
public class Person {
    private String name;
    private int age;
}

    204. Project Lombok-kal hogyan ad meg equals-t és HashCode-ot?

    • A @EqualsAndHashCode annotációval automatikusan generálható az equals és hashCode metódus.
    • Példa: 
      @EqualsAndHashCode
public class Person {
    private String name;
    private int age;
}

    205. A Jackson könyvtár segítségével milyen eszközök segítségével tud Stringbe illetve állományba írni JSON-t?

    • A Jackson ObjectMapper osztályának writeValueAsString() metódusa JSON-t generál String formátumban.
    • A writeValue(File file, Object obj) metódussal JSON-t írhatunk fájlba.

    206. A Jackson könyvtár segítségével milyen eszközök segítségével tud Stringből illetve állományból JSON-t olvasni?

    • A Jackson ObjectMapper osztályának readValue(String json, Class<T> valueType) metódusa JSON-t olvas Stringből.
    • A ObjectMapper.readValue(File file, Class<T> valueType) metódussal JSON-t olvashatunk fájlból.

    207. Soroljon fel és magyarázzon 3 Jackson annotációt!

    • @JsonProperty: Meghatározza, hogy egy mező milyen néven jelenjen meg a JSON-ben.
    • @JsonIgnore: Kizárja a megjelölt mezőt a JSON-ből íráskor vagy olvasáskor.
    • @JsonFormat: Meghatározza az idő- vagy dátumformátumot egy mezőnél (pl. "yyyy-MM-dd").

    208. Az előadásanyag a Java Annotáció 3 használatát említette, melyek voltak ezek?

    • Információ a fordító számára — Az annotációk használhatók a fordító által hibák észlelésére vagy figyelmeztetések elnyomására.
    • Fordítási idő és telepítési idő feldolgozása — Szoftvereszközök feldolgozhatják az annotációs információkat kód, XML fájlok stb. generálására.
    • Futásidejű feldolgozás — Egyes annotációk futásidőben is elérhetők és vizsgálhatók.

    209. Mi az az annotáció interfész és mire lehet használni?

    • Egy speciális interfész, amely metaadatokat biztosít a kódról (annotációkat definiál).
    • Például: meghatározhatjuk vele, hogy egy osztály vagy metódus milyen viselkedést kövessen (pl. tesztek jelölése).

    210. Pythonban hogy néz ki egy osztály konstruktora?

    • A Python konstruktor neve mindig: __init__().
    • Példa: 
      class MyClass:
    def __init__(self, name):
        self.name = name

    211. Pythonban hogy lehet egy osztály példányszintű és osztályszintű adattagjait definiálni?

    • Példányszintű adattag: Az objektum szintjén definiált attribútum (pl. self.name).
    • Osztályszintű adattag: Az osztály szintjén definiált attribútum (pl. class változó).

    212. Python hogy néz ki az öröklés osztályok és interfészek között?

    • Példa öröklésre: 
      class Parent:
    pass

class Child(Parent):
    pass
    • Interfészek: Pythonban nincs dedikált interfész fogalom, de egy osztály több osztályt is örökölhet, így több interfészhez hasonló struktúrát is képezhet.
